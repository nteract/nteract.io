## Headings and paragraph rhythm

Long-form content should feel like a natural extension of the rest of the site.
This paragraph is useful for checking measure, line height, spacing, and the way
links and inline code sit within body copy.

### Unordered, ordered, and task lists

- Top-level bullet items should align cleanly
- Nested content should still read comfortably
  - including a nested bullet
  - and an item with `inline code`
- Links inside lists should inherit the same styling as prose links

1. Ordered items should use the same spacing rhythm
2. Numbers should align clearly with multiline content
3. The list should remain readable on narrow layouts

- [x] GFM task lists should render as valid checklist content
- [ ] Pending items should stay readable even if we do not heavily customize them

#### TSX example

```tsx title="components/blog/example-card.tsx"
import Link from "next/link";

type ExampleCardProps = {
  title: string;
  href: string;
};

export function ExampleCard({ title, href }: ExampleCardProps) {
  return (
    <Link
      href={href}
      className="rounded-2xl border border-black/5 bg-white p-6 shadow-sm transition hover:-translate-y-0.5"
    >
      <h3 className="text-xl font-semibold tracking-tight text-gray-900">
        {title}
      </h3>
      <p className="mt-2 text-sm text-gray-600">
        A small reusable card for linking into long-form content.
      </p>
    </Link>
  );
}
```
